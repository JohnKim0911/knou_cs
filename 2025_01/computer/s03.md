## ✔️ 2. 수치 데이터의 표현과 진법변환

## ✔️ 3. 문자데이터의 표현방법

- 문자 데이터의 표현
  - 표현할 수 있는 수
    - 6비트: 64개 (2의 6승)
    - 7비트: 128개 (2의 7승)
    - 8비트: 256개 (2의 8승)
  - ASCII: 7비트 + 1비트(검증)
    - `A`merican `S`tandard `C`ode for Information Interchange
    - 미국 정보교환 표준코드로, 개인용 컴퓨터 및 데이터 통신용으로 가장 많이 사용
  - UNICODE: 16비트로 확장, 세계의 모든 언어들을 표현

- ASCII의 일부코드표

- UNICODE
  - 16비트: 65,536개 (2의 16승)
  - 세계의 모든 언어들을 표현

## ✔️ 4. 주기억장치 주소, 데이터 저장, 명령어

- 주기억장치 주소와 데이터 저장
  - 기억용량단위: bit, byte, word 등
    - 1Byte = 8 bits
    - 1KB = 1,024 Bytes
    - 1MB = 1,024 KB
    - 1GB = 1,024 MB
    - 1TB = 1,024 GB
    - 1PB = 1,024 TB
  - 호출시간 단위: 컴퓨터의 속도
    - 1msec (밀리초)
    - 1µsec (마이크로초) // 초당 100만 개의 자료를 호출한다
    - 1nsec (나노초)
    - 1psec (피코초)
  - 주기억장치의 각 바이트는 단일 주소를 갖는다.
  - 숫자, 영문자, 특수문자가 입력장치에서 읽혀 주기억장치로 보내지면, 비트의 형태로 바뀌어 기억된다.
  - 한 번 데이터가 수록되면 그 내용이 바뀔 때까지 남아 있게 된다.
  - 고급언어로 작성한 프로그램
    - 기계어로 번역
    - 주기억장치에 저장
    - 처리기 전자회로가 해석하고 실행.

- 기계어명령어의 형식
  - 명령부: 실행될 동작을 명시
  - 오퍼랜드부: 레지스터명칭, 데이터 주소, 데이터의 길이 등

## ✔️ 5. 주기억 장치의 발달과정

- 주기억장치의 발달과정
  - 진공관
  - 자기코어

- 반도체기억장치

- RAM(Random Access Memory)
  - SRAM(Static RAM): 
    - 전원이 공급되는 한 내용이 그대로 유지된다.
    - 캐시메모리(Cache Memory)로 이용됨.
  - DRAM(Dynamic RAM): 
    - 전원이 공급되더라도 내용의 소멸을 방지하기 위해 계속적으로 리프레싱(refreshing)을 해야 한다.
    - 주기억장치로 많이 이용됨.
  
- ROM(Read Only Memory)
  - ROM: 메모리가 제작될 때 데이터를 이 안에 기록한다.
  - PROM(Programmable ROM): 
    - ROM과 같은 기능
    - 이용자가 PROM을 조립해 넣기 전에 프로그램이나 데이터를 기록할 수 있다.

- 메모리 기술
  - 64k, 256M, 1Gb, …, 32Gb DRAM
  - 플래시메모리(flash memory): 비파괴 메모리 기술, 전력이 필요없는 비휘발성 메모리, 충격에 강함
  - 캐시메모리 : 빠르나 값이 비싸다.
